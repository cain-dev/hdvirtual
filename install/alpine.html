Instalação do Alpine

Introdução

Preparação da instalação

login

Assim que o sistema carregar vai aparecer a tela de login, logue como root:

        Welcome to Alpine Linux VERSION
        Kernel VERSION on an i686 / tty1

        localhost login: root

teclado

Para configurar o teclado use o comando:

setup-keymap

Determine o layout como br como no exemplo abaixo:

Select keyboard layout: br

Determine a variante como br como no exemplo abaixo:

Select keyboard variant: br

hostname

Para configurar o hostname use o comando:

setup-hostname seu-hostname

Configurar rede:

setup-interfaces -r

Vai retornar uma tela como a do exemplo abaixo:

Available interfaces are: eth0.
Enter '?' for help on bridges, bonding and vlans.
Which one do you want to initialize? (or '?' or 'done') [eth0]
Ip address for eth0? (or 'dhcp', 'none', '?') [dhcp]
Do you want to do any manual network configuration? (y/n) [n]
udhcpc: started, v1.37.0
udhcpc: broadcasting select for 192.168.1.100, server 192.168.1.1

Para o meu caso (usando conexão via cabo) eu só apertei Enter 3 vezes pois as opções padrão estavam ok, mas fique a vontade pra configurar.

SSH (opcional mas altamente recomendado)

Você pode fazer isso na unha, mas eu recomendo fortemente que você faça isso através de ssh. Antes de mais nada mude a senha do root com o comando:

passwd

Insira a nova senha e para configurar o serviço de ssh basta fazer como no exemplo abaixo:

setup-sshd

Which ssh server? ('openssh', 'dropbear' or 'none') [openssh]
Allow root ssh login? ('?' for help) [prohibit-password] yes
Enter ssh key or URL for root (or 'none') [none]
 * service sshd added to runlevel default
 * Caching service dependencies ...
ssh-keygen: generating new host keys: RSA ECDSA ED25519
 * Starting sshd ...

Perceba que eu usei yes para Allow root ssh login, como é lan eu vou logar com senha e tá tudo certo.

timezone

Vamos configurar a timezone com o comando:

setup-timezone

Caso já souber a sua timezone e quiser setar diretamente pode usar o comando:

setup-timezone America/Sao_Paulo

NTP

Vamos configurar o ntp com o comando:

setup-ntp busybox

Mirrors

Se quiser configurar os mirrors pode usar o comando abaixo, geralmente eu uso o da UFPR (que no momento dessa instalação é 36):

setup-apkrepos

Mirrors prontos? Atualize o cache com:

apk update

UDEV (opcional mas altamente recomendado)

Por padrão o Alpine usa o mdev que é mais do que suficiente para a esmagadora maioria dos casos, entretanto, se você quer setups mais complexos e modernos (como por exemplo, com uso de ZFS) mudar para udev é fundamental. Para instalar eudev use o comando:

apk add eudev

Agora você pode setar o sistema para usar udev com o comando:

setup-devd udev

Particionamento

Para não repetir toda a explicação de particionamento, formatação e montagem que eu fiz no guia de instalação do Arch eu vou deixar o link Passo 4 - Particionar as unidades de armazenamento. Nesse parte do tutorial vou deixar apenas os nomes das ferramentas no Alpine, siga até após a montagem das partições e antes do Passo 7.

  • As ferramentas cfdisk, fdisk e sfdisk já vem por padrão;

  • Instale o pacote gptfdisk para o comando gdisk e cgdisk;

  • Instale o pacote sgdisk para o comando sgdisk;

  • Instale o pacote parted para o comando parted;

  • Instale o pacote lsblk para o comando lsblk;

  • Instale o pacote zfs para a gerência completa de zfs;

  • Instale o pacote btrfs-progs para o comando mkfs.btrfs;

  • Instale o pacote wipefs para o comando wipefs;

  • Instale o pacote dosfstools para o comando mkfs.fat;

  • Instale o pacote exfatprogs para o comando mkfs.exfat;

  • Instale o pacote xfsprogs para o comando mkfs.xfs;

  • Instale o pacote e2fsprogs para o comando mkfs.ext4;

Lembre-se de carregar os módulos, como para modprobe btrfs e modprobe zfs!

Instalar o sistema

A alternativa do pacstrap para o Alpine é:

apk --arch x86 -X https://dl-cdn.alpinelinux.org/alpine/latest-stable/main/ -U --allow-untrusted --root /mnt --initdb add alpine-base linux-lts linux-firmware

Considere o uso do:

linux-firmware-none linux-firmware-rtl_nic

Habilitando repositório community:

sed -i 's/^#\(.*community\)/\1/' /etc/apk/repositories Habilitar repositório edge/community: echo "@edge-community $(grep -m1 'main' /etc/apk/repositories | sed -E 's|v[0-9.]+|edge|; s|main|community|')" >> /etc/apk/repositories Habilitar repositório edge/testing: echo "@edge-testing $(grep -m1 'main' /etc/apk/repositories | sed -E 's|v[0-9.]+|edge|; s|main|testing|')" >> /etc/apk/repositories

Atualizar cache:

apk update

Faça a fstab com o comando:

genfstab -U /mnt > /mnt/etc/fstab

Verifique com o comando:

cat /mnt/etc/fstab

Lembre-se de validar manualmente a fstab! E como corrigir flags como compress=zstd:1 ou remover os subvolid nas entradas.

resolv.conf

Para garantir o funcionamento da rede dentro do volume é interessante copiar o resolv.conf:

cp -L /etc/resolv.conf /mnt/etc/

chroot

Para preparar o chroot use os comandos:

mount -t proc none /mnt/proc mount -t sysfs none /mnt/sys mount -o bind /dev /mnt/dev mount -o bind /dev/pts /mnt/dev/pts

E para completar use o comando:

chroot /mnt /bin/sh

Mirrors

Se quiser configurar os mirrors pode usar o comando abaixo, geralmente eu uso o da UFPR (que no momento dessa instalação é 36):

setup-apkrepos

Mirrors prontos? Atualize o cache com:

apk update

pacotes base do sistema

apk add nano dosfstools e2fsprogs btrfs-progs kbd-bkeymaps tzdata iputils apk add intel/amd-ucode

Edite o init:

nano /etc/mkinitfs/mkinitfs.conf feature="...base btrfs..." mkinitfs

Configurações de hardware, relógio e periféricos misc

rc-update add hostname boot rc-update add hwclock boot rc-update add bootmisc boot

Módulos do kernel e parâmetros do sistema (sysctl.conf)

rc-update add modules boot rc-update add sysctl boot

Para garantir que o Alpine monte suas partições locais e ative a SWAP durante a inicialização:

rc-update add localmount boot rc-update add swap boot

timezone

Vamos configurar a timezone com o comando:

setup-timezone

Caso já souber a sua timezone e quiser setar diretamente pode usar o comando:

setup-timezone America/Sao_Paulo

hostname

Para configurar o hostname use o comando:

setup-hostname seu-hostname

teclado

Para configurar o teclado use o comando:

setup-keymap

Determine o layout como br como no exemplo abaixo:

Select keyboard layout: br

Determine a variante como br como no exemplo abaixo:

Select keyboard variant: br

Confirme a edição com o comando:

cat /etc/conf.d/loadkmap

rede

apk add networkmanager networkmanager-wifi networkmanager-tui networkmanager-cli wpa_supplicant dnsmasq rc-update add networkmanager default

passwd

Configure a senha do root com o comando:

passwd adduser -G wheel seu-usuario

doas

Para permitir elevação de privilégio recomendo o uso do doas, instale com:

apk add doas

Permitir o grupo wheel a elevação de privilégio, edite o arquivo /etc/doas.conf e habilite a linha:

# permit persist :wheel

Setando as permissões do arquivo:

chmod 600 /etc/doas.conf

gerenciamento de hardware

apk add eudev udev-init-scripts udev-init-scripts-openrc

Garante o escaneamento e reconhecimento do hardware no sysinit

rc-update add udev sysinit rc-update add udev-trigger sysinit rc-update add udev-settle sysinit rc-update add udev-postmount default

bootloader

Instalação dos pacotes:

apk add grub grub-efi efibootmgr

Instalação do grub:

grub-install --target=i386-efi --efi-directory=/efi --boot-directory=/boot --bootloader-id=GRUB --removable

Editar arquivo de configuração:

nano /etc/default/grub

Adicione a linha:

GRUB_CMDLINE_LINUX="modules=btrfs rootflags=subvol=@"

Gerar arquivo de configuração:

grub-mkconfig -o /boot/grub/grub.cfg

finalizando

saindo do chroot:

exit

desmontando

umount -l /mnt/dev/pts
umount -l /mnt/dev
umount /mnt/proc
umount /mnt/sys umount /mnt/efi
umount /mnt/boot reboot

Pós-instalação

fontes básicas

Para instalar o pacote básico de fontes use:

apk add ttf-liberation font-noto font-noto-cjk font-noto-emoji font-noto-extra

greetd

Para usar o greetd instale os pacotes:

doas apk add seatd greetd greetd-gtkgreet cage

Adicione seu usuário no grupo seat:

doas adduser seu_usuario seat

Adicione o greetd ao grupo seat:

doas adduser greetd seat

Edite o arquivo de configuração do greetd:

doas nvim /etc/greetd/config.toml 

[terminal]
vt = 7

[default_session]
command = "cage -s -- gtkgreet -l"
user = "greetd"

Adicione o serviço do seatd:

doas rc-update add seatd boot echo "rc_need=seatd" | doas tee -a /etc/conf.d/greetd

E adicione o serviço do greetd:

doas rc-update add greetd default
