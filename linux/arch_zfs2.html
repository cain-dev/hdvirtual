Instalando o Arch com raiz em ZFS v2

Dessa vez por eu mesmo e não por Sakamoto.

Introdução

Considerando que quase tudo em que o tutorial 1.0 foi baseado está descontinuado (fora o dkms) eu vou tentar criar um tutorial funcional já que os filhos da puta da página do Arch Linux Root on ZFS tão cagando pra documentação.

Preparando o sistema

Aqui vamos começar como está lá, mas com uma diferença: Baixar a versão mais recente para não ficar na mão!

Acesse a página Downloads do Alpine e baixe a versão Extended x86_64. Vamos fazer a preparação através dela.

Uma vez no sistema...

Assim que você tiver acessado o sistema e ele tiver carregado vai parar na tela de login:

localhost login:

Logue como root e o primeiro comando que você vai usar é o

setup-interfaces -r

Vai retornar uma tela como a do exemplo abaixo:

Available interfaces are: eth0.
Enter '?' for help on bridges, bonding and vlans.
Which one do you want to initialize? (or '?' or 'done') [eth0]
Ip address for eth0? (or 'dhcp', 'none', '?') [dhcp]
Do you want to do any manual network configuration? (y/n) [n]
udhcpc: started, v1.37.0
udhcpc: broadcasting select for 192.168.1.100, server 192.168.1.1

Para o meu caso (usando conexão via cabo) eu só apertei Enter 3 vezes pois as opções padrão estavam ok, mas fique a vontade pra configurar.

SSH (opcional)

Você pode fazer isso na unha, mas eu recomendo fortemente que você faça isso através de ssh, para configurar basta fazer como no exemplo abaixo:

setup-sshd

Which ssh server? ('openssh', 'dropbear' or 'none') [openssh]
Allow root ssh login? ('?' for help) [prohibit-password] yes
Enter ssh key or URL for root (or 'none') [none]
 * service sshd added to runlevel default
 * Caching service dependencies ...
ssh-keygen: generating new host keys: RSA ECDSA ED25519
 * Starting sshd ...

Perceba que eu usei yes para Allow root ssh login, como é lan eu vou logar com senha e tá tudo certo.

Engatilhando...

Vamos ativar o ntp com

setup-ntp busybox

Se quiser configurar os mirrors pode usar o comando abaixo, geralmente eu uso o ufpr (que no momento dessa instalação é 36):

setup-apkrepos

Mirrors prontos, atualize com:

apk update

Instalar eudev

apk add eudev

setar para usar udev

setup-devd udev

Você pode fazer o seu particionamento e formatação por aqui, mas eu vou presumir que você já tenha feito, só para fins de explicação os pactes que você pode precisar são:

  • cfdisk, fdisk, sfdisk (já vem por padrão)

  • gptfdisk para o comando gdisk e cgdisk

  • sgdisk para o comando sgdisk

  • parted para o comando parted

Só usar o comando abaixo para adicionar pacotes:

apk add nomedopacote

E o que não pode faltar de forma alguma para gerenciar esse arranjo de ZFS:

apk add zfs util-linux modprobe zfs

Arquitetura de partições do sistema

Existem diversas maneiras de fazer os arranjos, nos meus sistemas eu geralmente faço uma partição de boot (XBOOTLDR) separada e não utilizo o GRUB2, portanto, não tenho gerência do ZFS na partição /boot.

Caso você tenha uma partição XBOOTLDR padrão e não queira usar gerência, basta montar ela em /mnt/boot. Caso queira uma partição de boot com ZFS e compatível com tal gerência basta seguir a documentação e usar a configuração de bpool como no exemplo abaixo.

Nada de SDA!

Sempre que for fazer algo baseado em ZFS, dê preferência a utilizar ID do disco/partição ao invés de qualquer outro meio, para listar as partições disponíveis use:

find /dev/disk/by-id/

rpool (piscina da raiz do sistema)

Para criar a piscina de dados para a rpool use o comando:

zpool create \
-o ashift=12 \
-R /mnt \
-O acltype=posixacl \
-O canmount=off \
-O dnodesize=auto \
-O normalization=formD \
-O relatime=on \
-O xattr=sa \
-O compression=zstd-3 \
-O mountpoint=none \
rpool \
/dev/disk/by-id/ID_DO_DISCO-partNR_DA_PARTIÇÃO_ROOT

AVISO!

Sinta-se a vontade para utilizar -O atime=off ao invés -O relatime=on de para remover o access time, -O compression=off caso não queira compressão nessa partição e -o autotrim=on para habilitar o trim automático (útil para memórias flash).

bpool (piscina do boot do sistema)

Para criar a piscina de dados para a bpool use o comando:

zpool create \
-o ashift=12 \
-R /mnt \
-o compatibility=grub2 \
-O dnodesize=legacy \
-O devices=off \
-O acltype=posixacl \
-O xattr=sa \
-O compression=lz4 \
-O normalization=formD \
-O relatime=on \
-O canmount=off \
-O mountpoint=none \
bpool \
/dev/disk/by-id/ID_DO_DISCO-partNR_DA_PARTIÇÃO_BOOT

AVISO!

Mais uma vez: Sinta-se a vontade para utilizar -O atime=off ao invés -O relatime=on de para remover o access time, -O compression=off caso não queira compressão nessa partição e -o autotrim=on para habilitar o trim automático (útil para memórias flash).

dpool (piscina de dados do sistema)

Data Pool ou dpool é aquele arranjo preparado com os arquivos de dados para além do sistema, seja para caso você tenha um disco que queira usar como /home ou para coisas que podem ser desacoplados de um sistema específico para serem usados fora, como por exemplo imagens em /var/lib/libvirt.

Para criar a piscina de dados para a dpool use o comando:

zpool create \
-o ashift=12 \
-R /mnt \
-O acltype=posixacl \
-O canmount=off \
-O dnodesize=auto \
-O normalization=formD \
-O relatime=on \
-O xattr=sa \
-O compression=zstd-3 \
-O mountpoint=none \
dpool \
/dev/disk/by-id/ID_DO_DISCO-partNR_DA_PARTIÇÃO_HOME

AVISO!

E mais uma última vez: Sinta-se a vontade para utilizar -O atime=off ao invés -O relatime=on de para remover o access time, -O compression=off caso não queira compressão nessa partição e -o autotrim=on para habilitar o trim automático (útil para memórias flash).

Criando datasets

Entenda o dataset como uma "partição virtual", é mais complexo que isso, mas é como se fosse uma partição com pontos de montagem e atributos do próprios. Isso aqui é só uma média, fique a vontade para alterar ou mesmo não fazer algum desses se você souver o que está fazendo.

Conteiners principais

zfs create -o canmount=off -o mountpoint=none rpool/ROOT

Se e somente se fez a bpool

zfs create -o canmount=off -o mountpoint=none bpool/BOOT

Ponto de montagem para a raiz e boot

zfs create -o canmount=noauto -o mountpoint=legacy rpool/ROOT/arch

Se e somente se fez a bpool

zfs create -o mountpoint=legacy bpool/BOOT/arch

/home

zfs create -o setuid=off -o devices=off -o mountpoint=legacy rpool/home

/root

zfs create -o devices=off -o mountpoint=legacy rpool/root

/var

zfs create -o canmount=off -o mountpoint=none rpool/var zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/log zfs create -o com.sun:auto-snapshot=false -o compression=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/cache zfs create -o com.sun:auto-snapshot=false -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/tmp

/var/lib

zfs create -o canmount=off -o mountpoint=none rpool/var/lib zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/lib/AccountsService

/usr

zfs create -o canmount=off -o mountpoint=none rpool/usr zfs create -o devices=off -o mountpoint=legacy rpool/usr/local

/opt

zfs create -o devices=off -o mountpoint=legacy rpool/opt

/tmp

Um alerta rápido sobre o /tmp, tanto você pode simplesmente deixar esse diretório na memória (maioria dos casos), ou dependendo do teu uso (talvez compilação) você pode fazer uso de mais espaço do que a sua memória ram. Depende do uso de arquivos temporários na sua máquina e o quanto de ram você tem.

zfs create -o com.sun:auto-snapshot=false -o setuid=off -o devices=off -o mountpoint=legacy rpool/tmp

Datasets de função específica (opcionais)

/var/lib/docker

Armazena imagens, containers e volumes do Docker. Cresce muito rapidamente e gerencia próprios snapshots internamente. Fora do snapshot do sistema para não reverter dados de containers junto com o OS.

zfs create -o com.sun:auto-snapshot=false -o compression=zstd -o mountpoint=legacy rpool/var/lib/docker

/var/lib/libvirt

Armazena configurações e imagens de máquinas virtuais. Imagens de VM podem ocupar dezenas de GB e gerenciam próprios snapshots. Fora do snapshot do sistema para não reverter VMs junto com o OS.

zfs create -o canmount=off -o mountpoint=none rpool/var/lib/libvirt zfs create -o com.sun:auto-snapshot=false -o compression=off -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/lib/libvirt/images

/var/www

Armazena arquivos de sites e aplicações web servidos pelo servidor HTTP. Dados do site têm ciclo de vida independente do OS e não devem reverter junto.

zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/www

/var/lib/nfs

Armazena estado de bloqueios e tabelas de montagem do servidor NFS. Estado de rede não deve reverter com o OS pois causaria inconsistência nos clientes.

zfs create -o com.sun:auto-snapshot=false -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/lib/nfs

/var/lib/samba

Armazena senhas, configurações de compartilhamento e estado de conexões do Samba. Estado de rede não deve reverter com o OS pois causaria perda de configurações de compartilhamento.

zfs create -o com.sun:auto-snapshot=false -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/lib/samba

/var/spool

Armazena filas de impressão, cron e outros serviços de fila. Em desktop raramente relevante, mas vale separar se usar impressora ou serviços de fila intensivamente.

zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/spool

/srv

Armazena dados servidos por serviços como FTP e HTTP. Convenção do FHS para dados de serviços — ciclo de vida independente do OS.

zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/srv

/var/mail

Armazena emails locais entregues por MTAs como Sendmail ou Postfix. Emails têm ciclo de vida independente do OS e não devem reverter junto. Não relevante para clientes de email como Thunderbird.

zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/mail

/var/snap

Armazena pacotes e dados de aplicações instaladas via Snap. No Arch Linux raramente usado — AUR e Flatpak são preferidos.

zfs create -o compression=off -o mountpoint=legacy rpool/var/snap

/var/games

Armazena pontuações e dados de jogos instalados via gerenciador de pacotes. Diretório legado do FHS — jogos modernos como Steam usam diretórios próprios.

zfs create -o exec=off -o setuid=off -o devices=off -o mountpoint=legacy rpool/var/games

Monte as demais partições

swap

Se você fez uma partição SWAP, ou vai usar ESP ou qualquer que seja, monte elas agora.

swapon /dev/storageAP2
Usando TRIM na partição SWAP (Clique para ver)

Ao contrário das demais partições onde o serviço fstrim.timer semanal resolve tudo sozinho, o fstrim ignora a partição swap completamente.

Se você quiser montar a partição com o trim no carregamento você pode usar:

swapon --discard=once /dev/storageAP2

Se você quiser montar a partição com algo mais próximo do trim contínuo você pode usar:

swapon --discard=pages /dev/storageAP2

esp

mount -o fmask=0077,dmask=0077,iocharset=iso8859-1,X-mount.mkdir /dev/storageAP1 /mnt/efi

Setando as permissões

chmod 700 /mnt/root chmod 1777 /mnt/tmp chmod 1777 /mnt/var/tmp

Preparando o bootstrap

Preparando cachefile

Para livrar de algumas dores de cabeça, é interessante preparar o arquivo de cache antes do bootstrap:

zpool set cachefile=/etc/zfs/zpool.cache rpool

Se e somente se fez uma bpool

zpool set cachefile=/etc/zfs/zpool.cache bpool mkdir -p /mnt/etc/zfs cp /etc/zfs/zpool.cache /mnt/etc/zfs/zpool.cache

Preparando o pacote

apk add curl curl --fail-early --fail -L https://archlinux.c3sl.ufpr.br/iso/latest/archlinux-bootstrap-x86_64.tar.zst -o rootfs.tar.zst
(Opcional porém recomendado) Verificar assinatura (Clique para ver) apk add gnupg curl --fail-early --fail -L https://archlinux.c3sl.ufpr.br/iso/latest/archlinux-bootstrap-x86_64.tar.zst.sig -o rootfs.tar.zst.sig gpg --auto-key-retrieve --keyserver hkps://keyserver.ubuntu.com --verify rootfs.tar.zst.sig rootfs.tar.zst

Fazendo o strap

Agora que temos o pacote preciamos do suporte ao zstd:

apk add zstd

E agora fazer o strap:

zstd -dc rootfs.tar.zst | tar xf - -C /mnt --strip-components=1

Verifique se o boostrap funcionou com:

ls -p /mnt

Deve resultar em:

bin boot/ dev/ etc/ home/ lib lib64 mnt/ opt/ proc/ root/ run/ sbin srv/ sys/ tmp/ usr/ var/

Habilitando o repositório community e arch-install-scripts

Habilite o repositório com os comandos:

sed -i '/edge/d' /etc/apk/repositories sed -i -E 's/#(.*)community/\1community/' /etc/apk/repositories

Instale o arch-install-scripts com o comando:

apk add arch-install-scripts

Prepare a fstab com o comando:

genfstab -t PARTUUID /mnt | grep -v swap | sed "s|vfat.*rw|vfat rw,x-systemd.idle-timeout=1min,x-systemd.automount,noauto,nofail|" > /mnt/etc/fstab

Verifique com o comando:

cat /mnt/etc/fstab

Fazendo o chroot

cp /etc/resolv.conf /mnt/etc/resolv.conf for i in /dev /proc /sys; do mkdir -p /mnt/"${i}"; mount --rbind "${i}" /mnt/"${i}"; done chroot /mnt /usr/bin/env bash

Preparando o repositório ArchZFS

pacman-key --init pacman-key --refresh-keys pacman-key --populate pacman-key --recv-keys 3A9917BF0DED5C13F69AC68FABEC0A1208037BE9 pacman-key --lsign-key 3A9917BF0DED5C13F69AC68FABEC0A1208037BE9 echo -e "\n[archzfs]\nSigLevel = Required\nServer = https://github.com/archzfs/archzfs/releases/download/experimental" >> /etc/pacman.conf sed -i '/c3sl.ufpr.br/s/^#//' /etc/pacman.d/mirrorlist pacman -Sy kernel_compatible_with_zfs="$(pacman -Si zfs-linux | grep 'Depends On' | sed 's|.*linux=||' | awk '{print $1}')" pacman -U --noconfirm https://archlinux.c3sl.ufpr.br/core/os/x86_64/linux-"${kernel_compatible_with_zfs}"-x86_64.pkg.tar.zst pacman -S --noconfirm zfs-linux zfs-utils linux-firmware

Instale o seu editor de texto de preferência, para este tutorial vou usar o nano:

pacman -S nano

Edite o /etc/mkinitcpio.conf, e insira o módulo zfs:

MODULES=(zfs)

E o hook zfs:

HOOKS=(... zfs filesystems ...)

Vamos gerar o hostid:

zgenhostid -f -o /etc/hostid

Configurando a instalação

Para que isso fique dedicado as alterações do ZFS peço para que você vá documentação da instalação do Arch e siga do Passo 10 - Configurar o sistema instalado até a instalação do ucode no Passo 13 - Instalação de inicializador.


Instale normalmente o seu bootloader, mas lembre-se de adicionar o parâmetro abaixo para apontar para o volume em ZFS:

root=ZFS=rpool/ROOT/arch rw

No caso do GRUB seria como no exemplo abaixo:

GRUB_CMDLINE_LINUX="root=ZFS=rpool/ROOT/arch rw"

Setar a raiz

Este comando básicamente diz "até que se diga o contrário, busque a raiz aqui":

zpool set bootfs=rpool/ROOT/arch rpool

Criando o initramfs:

mkinitcpio -P

Habilitar serviços do systemd

systemctl enable zfs.target systemctl enable zfs-import-cache.service systemctl enable zfs-import.target

Finalizando a instalação

Antes de mais nada, saia do ambiente chroot:

exit

Criar snapshots da instalação original:

zfs snapshot -r rpool@initial

Se e somente se você tem bpool:

zfs snapshot -r bpool@initial

Agora, desmontar todas partições:

umount /mnt/efi umount /mnt/boot swapoff -a

Exportar a rpool:

zpool export rpool

Se tiver a bpool:

zpool export bpool

Reiniciar a máquina:

reboot
